## Развертывание и восстановление проекта

### Шаг 1. Подготовка файлов

1. Скачайте файлы бэкапа (архивы `.tar.gz`, `.tar.gz.1` и т.д.).
2. Скачайте актуальный скрипт восстановления **`restore.php`** с официального сайта 1С-Битрикс.
3. Полностью очистите корневую директорию сайта.
4. Загрузите туда файлы бэкапа и `restore.php`.

### Шаг 2. Запуск веб-интерфейса

1. Откройте в браузере: `http://ваш-домен.ру/restore.php`
2. В мастере выберите вариант: **«Архив загружен в корневую папку сайта»**.
3. Дождитесь окончания распаковки архива.
4. Введите доступы к созданной базе данных (Сервер, Имя БД, Логин, Пароль).
5. После успешного завершения установки нажмите кнопку **«Удалить локальный резервный архив и служебные скрипты»**.

## Запуск импорта товаров

Скрипт импорта реализован в виде консольного скрипта запускаемого через Cron.

### Путь к скрипту

```bash
/local/cron/import_1c.php
```

### Пример файла импорта

```bash
/upload/1c_import/import_example.xml
```

### Варианты запуска

#### 1. Ручной запуск через консоль (SSH)

Для проверки работы скрипта перейдите в корневую папку сайта и выполните команду:

```bash
php -f ./local/cron/import_1c.php
```

#### 2. Автоматический запуск через Cron

Для регулярного обновления товаров добавьте задачу в планировщик `crontab -e` (пример для корневой директории сайта
/home/bitrix/site/):

```bash
0 1 * * * /usr/bin/php -f /home/bitrix/site/local/cron/import_1c.php > /dev/null 2>&1
```

## Кэширование на странице каталога

Для страницы каталога настроено встроенное управляемое кэширование Bitrix со следующими параметрами:

* `"CACHE_TYPE" => "A"` (Автоматический режим)
* `"CACHE_TIME" => "86400"` (Время жизни кэша — 24 часа)

### Как работает кэш

1. **Первый запрос**: Компонент делает запросы к базе данных, собирает данные, генерирует HTML-код и сохраняет результат
   в файл кэша на сервере.
2. **Повторные запросы**: Bitrix не обращается к базе данных, а мгновенно отдает пользователю готовый HTML-код из файла.
3. **Уникальность**: Кэш создается отдельно для каждого состояния страницы. Система учитывает примененные фильтры,
   сортировку и постраничную навигацию.

### События сброса кэша

Кэш автоматически сбрасывается в следующих случаях:

* **Истечение времени**: Прошло 24 часа с момента создания файла кэша.
* **Изменение контента**: В панели управления или через API был добавлен, изменен или удален товар, раздел или цена.
* **События инфоблоков**: Сработали стандартные механизмы `Managed Cache` при обновлении инфоблока каталога.
* **Ручной сброс**: Администратор очистил кэш вручную.
